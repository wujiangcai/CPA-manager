# 项目功能审计与优化记录

审计范围：账号池监控、CPA/Sub2 批量验活、总消耗代理监控、合并面板、启动配置与现有自动化测试。

## 已修复的高优先级问题

### 1. 未配置 auth_dir 时可能扫描当前目录

`Path("")` 会指向当前工作目录。旧逻辑在 `auth_dir` 为空时仍扫描 `*.json`，强制清理时理论上可能把配置 JSON 当成无效账号处理。

现在空 `auth_dir` 会立即返回明确警告，不再扫描或清理当前目录；空的 `cpa_config_path` 也不会再尝试读取当前目录。

### 2. 未刷新额度时可能把新导入账号归入 dead

当 API 使用 `refresh=0` 且缓存不能覆盖新文件时，旧流程仍继续执行分类，账号会因 `quota_missing` 进入 `dead`。

现在只有缓存新鲜且完整覆盖本批账号时才允许继续；否则停止归档和删除，账号仍保留在本地号池，避免网络或缓存问题造成误判。

### 3. cleanup_quarantine_dir 配置未生效

旧清理逻辑始终调用 `unlink()`，配置中的隔离目录没有被使用，返回值中的 `moved` 也始终为 0。

现在配置隔离目录后，会按清理原因移动账号文件，并处理同名冲突；仅在隔离目录为空时永久删除。

### 4. 导入请求缺少体积和完整性校验

批量导入曾直接按照任意 `Content-Length` 读取内存，并且没有判断客户端是否提前断开。

现在新增 `import_max_bytes`，浏览器和服务端都会校验上限；服务端还会拒绝不完整请求体，避免内存异常和截断 JSON。

### 5. 配置类型和时间戳兼容性

- 字符串 `"false"` 曾会被 Python `bool()` 当成 true；现在布尔配置支持常见字符串形式。
- 非法端口、并发、超时配置曾可能让服务启动或状态接口直接报错；现在统一使用带默认值和上下限的解析器。
- 毫秒级 Unix 过期时间曾无法解析；现在自动识别秒/毫秒时间戳。

### 6. 总消耗监控的凭证和代理健壮性

- 旧账号标识直接保存 Bearer Token 最后 8 位；现在只保存 SHA-256 摘要前 12 位。
- 请求体提前断开时旧逻辑可能把截断数据转发上游；现在返回 400。
- 上游已经开始流式响应后发生超时或异常时，旧逻辑可能再写第二个 HTTP 错误响应；现在只关闭当前连接。
- `BrokenPipeError` 不再被错误包装成上游 502。

### 7. 合并面板和号池页面问题

- 合并面板 iframe 与 ping 地址不再写死，支持环境变量配置，并对 HTML 属性转义。
- 修复自动清理相关乱码日志。
- 删除页面中无法执行的重复 JavaScript。
- 低额度按钮和确认文案不再写死 5%，会读取实际阈值。
- 查询额度和立即清理改用 POST，避免用 GET 触发修改状态的操作。

## 新增：验活后选择账号提交远端号池

完整流程：

1. 导入 CPA/Sub2 JSON 并执行真实 WHAM 验活；
2. 页面列出最近批次所有 `alive` 账号；
3. 用户全选、取消全选或逐个勾选；
4. 服务端重新校验批次、文件名、归档路径、账号类型和 Access Token；
5. 并发提交到远端 CPA `/v0/management/auth-files`；
6. 持久化每个账号的上传成功、失败、时间和错误，支持失败重试。

服务端只接受最近验活批次中的 `alive` 文件，不接受 `dead` 文件、任意磁盘路径或目录穿越文件名。

## 验证覆盖

自动化测试包含：

- CPA 与 Sub2 格式识别、规范化和 WHAM 验活；
- Sub2 完整批次导入、存活归档；
- 选中账号远端上传、Management API URL、Bearer 管理密钥和 HTTP 路由；
- 非 alive 与目录穿越拒绝；
- 无缓存时停止死亡归档；
- 清理隔离目录；
- 空 auth_dir、毫秒时间戳和配置容错；
- Token 标识哈希、截断请求体；
- 合并面板可配置 URL 和 HTML 转义；
- 内嵌页面 JavaScript 语法检查。

## 仍需注意的部署边界

- 监控面板自身没有登录系统，默认只监听 `127.0.0.1`。如需公网访问，应放在带鉴权的反向代理后面。
- 远端提交当前目标是 CPA Management API，不是 sub2api 管理 API；Sub2 输入会先转换成 CPA auth。
- 远端管理密钥建议只通过 `CPA_REMOTE_POOL_MANAGEMENT_KEY` 环境变量提供。
- 金额统计取决于 `model_prices.json`，新增模型后仍需维护价格表。
