# zny CPA 监测系统使用教程

QQ 交流群：454765232

这个项目是一个本地 CPA 监测面板，主要包含两部分：

- CPA 总消耗监控：统计请求数、token、模型、耗时、首字时间、估算金额。
- CPA 号池监测：读取本机 CPA auth 目录，查看账号 Plan、额度、失效状态，并支持清理失效账号。

## 1. 准备环境

需要 Windows + Python 3.10 或更高版本。

在 CMD 或 PowerShell 里检查：

```powershell
python --version
```

如果提示找不到 Python，先安装 Python，并勾选 Add Python to PATH。

## 2. 解压项目

把压缩包解压到一个固定目录，例如：

```text
D:\zny-cpa-monitor
```

目录结构大概是：

```text
zny-cpa-monitor
├─ RUN_ON_YOUR_PC
├─ account_pool_monitor
├─ start_all.cmd
├─ stop_all.cmd
├─ start_all.ps1
└─ stop_all.ps1
```

## 3. 配置 CPA 总消耗监控

进入 `RUN_ON_YOUR_PC`，复制配置文件：

```powershell
copy monitor_config.example.json monitor_config.json
```

打开 `monitor_config.json`，重点看这几个字段：

```json
{
  "upstream_base_url": "http://127.0.0.1:8317/v1",
  "upstream_api_key": "",
  "upstream_api_key_env": "CPA_MONITOR_UPSTREAM_API_KEY",
  "preserve_client_authorization": true
}
```

说明：

- `upstream_base_url`：你的 CPA 地址，一般是 `http://127.0.0.1:8317/v1`。
- `upstream_api_key`：如果你的 CPA 本机需要 key，可以填这里。
- `upstream_api_key_env`：也可以不把 key 写进文件，而是用环境变量。
- `preserve_client_authorization`：如果前端请求自带 Authorization，就保留原来的 key。

推荐用环境变量放 key：

```powershell
$env:CPA_MONITOR_UPSTREAM_API_KEY="你的CPA本地key"
```

## 4. 配置 CPA 号池监测

进入 `account_pool_monitor`，复制配置文件：

```powershell
copy monitor_config.example.json monitor_config.json
```

打开 `monitor_config.json`，重点改：

```json
{
  "auth_dir": "D:/YOUR_CPA/auth",
  "cpa_base_url": "http://127.0.0.1:8317",
  "quota_query_mode": "direct_auth"
}
```

说明：

- `auth_dir`：你的 CPA auth 账号 JSON 目录。
- `cpa_base_url`：CPA 本地管理地址，通常是 `http://127.0.0.1:8317`。
- `quota_query_mode`：默认 `direct_auth`，会直接读取 auth JSON 里的 token 查询额度。

注意：不要把自己的 auth 目录、token、真实配置发给别人。

## 7. 怎么接入 CPA 总消耗监控

如果你想统计所有 CPA 请求，需要让请求先经过总消耗监控，再转发到 CPA。

原 CPA 地址：

```text
http://127.0.0.1:8317/v1
```

改成监控地址：

```text
http://127.0.0.1:18319/v1
```

客户端 Base URL 填：

```text
http://127.0.0.1:18319/v1
```

监控系统会记录请求后，再转发给真正的 CPA。

## 8. 号池清理说明

号池面板支持识别并清理这些账号：

- 401：认证失效
- 402：账号、工作区不可用或没额度
- 403：无权限
- 404：账号不可用
- 503：服务侧返回不可用
- exhausted：额度耗尽
- expired：账号过期
- missing token：缺 token

低于 5% 额度的账号有单独开关：

- 开启低于 5% 删除：低额度账号会被清理。
- 关闭低于 5% 删除：低额度账号只显示，不自动删，也不被立即清理删。

手动禁用的账号默认会跳过清理。

为了避免误删，可在 `account_pool_monitor/monitor_config.json` 设置隔离目录：

```json
{
  "cleanup_quarantine_dir": "D:/CPA_QUARANTINE"
}
```

配置后，自动清理会把失效 JSON 按原因移动到隔离目录；留空才会直接删除。

## 9. 验活后提交远端号池

在“CPA / Sub2 JSON 一键验活”完成后，页面会显示本批次存活账号列表：

1. 勾选要提交的账号；
2. 可使用“全选存活”或“取消全选”；
3. 点击“提交选中账号”；
4. 页面会记录每个账号的成功、失败及上传时间。

先配置远端 CPA：

```json
{
  "remote_pool_base_url": "https://YOUR_REMOTE_CPA_HOST",
  "remote_pool_management_key": "",
  "remote_pool_upload_concurrency": 16,
  "remote_pool_timeout_seconds": 25
}
```

推荐使用环境变量保存远端管理密钥：

```powershell
$env:CPA_REMOTE_POOL_MANAGEMENT_KEY="你的远端CPA管理密钥"
```

程序只允许提交最近一次验活批次中的 `alive` 账号，不允许选择 `dead` 文件。

## 10. 常见问题

### 页面打不开

先确认是否启动成功：

```powershell
curl http://127.0.0.1:18321/api/ping
```

如果没有响应，重新运行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\start_all.ps1
```

### 查询额度没反应

检查 `account_pool_monitor/monitor_config.json` 里的 `auth_dir` 是否正确。

### 金额不准

金额是按 `RUN_ON_YOUR_PC/model_prices.json` 计算的。不同 CPA 的模型名可能不同，需要自己改价格表。

### 不要开源哪些文件

不要上传这些：

```text
monitor_config.json
monitor_data/
*.log
*.sqlite3
你的 CPA auth 目录
任何 token / key / 密钥文件
```
